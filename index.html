🎯 실전 프로젝트 과제 비교 분석

4인 팀, 35시간(1주일) 기준

1️⃣ AI-Human 판별
난이도: ⭐⭐ (2/5)
✅ 장점
• End-to-end 구성이 간단함 (텍스트 분류 기본 문제)
• 명확한 이진 분류 문제로 평가 메트릭 정의 용이
• 학습 데이터, 검증용 보조 데이터 모두 충분함
• 빠르게 베이스라인 모델 구축 가능
• 팀원이 각자 다른 모델 시도 가능 (병렬 작업)
❌ 단점
• 강건성 유지가 핵심 제약 사항 (보조 데이터와의 성능 격차 최소화 필요)
• 단순 학습만으로는 부족하고 도메인 적응 기법 필요
• 발표에서 차별성 있는 개선안 제시 어려울 수 있음
⚙️ 기술 스택
BERT/RoBERTa 파인튜닝, 텍스트 전처리, 클래스 불균형 처리
👥 팀 분담 예시
• 1명: EDA 및 데이터 전처리
• 1명: 기본 모델 구축 (BERT)
• 1명: 도메인 적응 기법 연구 (Data augmentation, Domain adversarial)
• 1명: 검증 및 발표 자료 준비
추천 대상: 첫 프로젝트 경험이 부족한 팀
2️⃣ 긴급도 판별
난이도: ⭐⭐⭐ (3/5)
✅ 장점
• 멀티모달(음성+텍스트) + 멀티태스크(긴급도+감정) 으로 포트폴리오 가치 높음
• 실무 문제와 유사한 현실적인 과제
• 공식 데이터 (AIHub) 통해 신뢰성 높음
• 멀티태스크 학습 시 시너지 효과 가능
❌ 단점
• 음성 처리 (음향 특징 추출, 음성 모델) 학습 필요
• 텍스트 STT 변환 오류 처리 필요
• 두 모달리티 융합 방법 설계 필요 (Early/Late/Hybrid fusion)
• 데이터 크기가 크면 전처리 시간 소요
• 모델 학습 시간이 상대적으로 길 수 있음
⚙️ 기술 스택
음성 전처리 (Librosa), 음향 특징 추출, 텍스트 임베딩, 멀티모달 융합, 멀티태스크 러닝
👥 팀 분담 예시
• 1명: 음성 데이터 전처리 및 음향 특징 추출
• 1명: 텍스트 전처리 및 임베딩
• 1명: 멀티모달 모델 아키텍처 설계 및 구현
• 1명: 멀티태스크 손실 함수 조정, 검증, 발표 자료
추천 대상: AI/음성 처리 경험 있는 팀
3️⃣ 발화자 감정 판별
난이도: ⭐⭐⭐ (3/5)
✅ 장점
• 텍스트 기반으로 구현 난이도 낮음
• 멀티태스크 학습 (불안 2개, 우울 2개 = 4개 예측)
• 회귀 + 분류 문제 조합으로 다양한 학습 경험
• 대화 컨텍스트 활용으로 고급 기법 시도 가능
❌ 단점
• 핵심 문제: 불안/우울 점수가 주관적이고 매우 불완전한 데이터
• 라벨 노이즈가 심할 수 있어 성능 향상에 한계
• 고령자 경험 기반 데이터로 도메인 특화성 높음
• 데이터 품질 검증에 시간 소요
• 강건한 모델 학습이 어려움
⚙️ 기술 스택
시퀀스 모델 (LSTM/Transformer), 멀티태스크 러닝, 회귀 + 분류 혼합, 라벨 스무딩
👥 팀 분담 예시
• 1명: 데이터 품질 분석 및 라벨 검증
• 1명: 텍스트 전처리 및 회귀 기준 설정
• 1명: 멀티태스크 모델 구현
• 1명: 라벨 노이즈 처리 기법 연구, 검증
주의: 라벨 품질 이슈로 성능 향상 제한적
4️⃣ AI 품질 평가
난이도: ⭐⭐ (2/5)
✅ 장점
• 9개 독립 이진 분류로 명확한 문제 정의
• 구현 난이도 낮음 (멀티레이블 분류)
• 9개 모델을 병렬로 학습 가능해 팀 분담 좋음
• 데이터가 상대적으로 깔끔할 가능성 높음
• 빠르게 기본 모델 완성 가능
❌ 단점
• 9개 독립 분류로 개선 전략 복잡함
• 클래스 불균형 가능성 높음
• 평가 기준(예: "흥미성")이 주관적이어서 데이터 품질 의존도 높음
• 최종 성능은 가장 낮은 클래스의 F1 스코어에 의존
• 발표에서 차별화된 인사이트 제시 어려움
⚙️ 기술 스택
멀티레이블 분류, 클래스 불균형 처리 (가중치 조정, SMOTE), 앙상블 기법
👥 팀 분담 예시
• 1명: 데이터 전처리 및 클래스 불균형 분석
• 2명: 각각 4~5개 클래스의 개별 모델 구축
• 1명: 앙상블 방법, 메타 모델 설계, 발표 준비
추천 대상: 제한된 시간으로 빠르게 완성하고 싶은 팀
5️⃣ 난독화 복원
난이도: ⭐⭐ (2/5)
✅ 장점
• 구현 난이도 낮음 (시퀀스 투 시퀀스 또는 BERT 마스킹)
• 한글 특화 문제로 한국 NLP 경험 적립
• 한글 음절 단위 처리로 토큰화 전략 구현 경험
• 경진대회 데이터로 신뢰성 높음
❌ 단점
• 데이터 크기가 작음 → 충분한 학습 데이터 부족
• 작은 데이터로 높은 성능 달성 어려움
• Transfer learning 없으면 오버피팅 위험
• 한글 특화로 인한 전이성 제한 (포트폴리오 가치)
• 시간 대비 성과 기대 낮음
⚙️ 기술 스택
한글 토큰화, Seq2Seq, BERT 마스킹, 한글 사전학습 모델 (KoBERT 등)
👥 팀 분담 예시
• 1명: 한글 전처리 및 데이터 증강
• 1명: 사전학습 모델 활용 (KoBERT/KoGPT)
• 1명: Seq2Seq 또는 생성 모델 구현
• 1명: 검증 및 발표
주의: 데이터 크기 부족 → 성능 향상 어려움
6️⃣ 코드 유사도 판별
난이도: ⭐⭐⭐⭐ (4/5)
✅ 장점
• 사전학습(Pretrained) 모델 구축으로 고급 기법 경험
• 대규모 보조 데이터 (Project CodeNet) 활용
• 파인튜닝 성능 개선 효과 명확함
• 포트폴리오 가치가 매우 높음
• 소프트웨어 엔지니어링 역량 입증
❌ 단점
• 코드 임베딩 모델 구축 난이도 높음
• 사전학습 시간 매우 오래 걸림 (GPU 필수)
• Project CodeNet 데이터 전처리 복잡
• 1주일 안에 충분한 사전학습 어려움
• 조기에 시작하지 않으면 실패 가능성 높음
• 코드 분석 및 유사도 메트릭 설계 필요
⚙️ 기술 스택
코드 파싱, AST(추상 구문 트리) 분석, 그래프 신경망, 코드 임베딩 (GraphCodeBERT), 전이학습
👥 팀 분담 예시
• 1명: Project CodeNet 데이터 전처리 및 코드 파싱
• 1명: 코드 임베딩 모델 구축 (GraphCodeBERT 또는 유사 모델)
• 1명: 사전학습 수행 및 모니터링
• 1명: 파인튜닝 및 평가
⚠️ 매우 도전적 - 숙련된 팀 권장, 초기 착수 필수
